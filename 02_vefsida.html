Project 1

Preparation

I have no prior experience in HTML or CSS programming so I decided to use a template from the website html5up.net that I could build on. The template I chose is called Editorial, the link to it can be found here . The reason why I chose this template was because I liked the side banner it provided.



When I had downloaded this template I saved it locally to my computer and opened it with Visual Studio Code. I use VSCode for all of my coding projects and I wanted to do the same with my future HTML projects. To make VSCode easy to use for HTML I downloaded the Live Preview Extension to VSCode. This extension lets you preview your work in a browser and updates the website automatically while you work on it.

Personalizing the website

As I've mentioned, I had no former experience using HTML so before I started working on the project I read through this article from w3schools to understand the basics of HTML. Then I started personalizing the template.

Formatting

Here are some things I did to format the text and images on the website

Adding Images

I wanted to add a border to the images and have the option of scaling them. I did that using the following code: 

<img src = "images/VSCode.png"
	style="object-fit:contain;
		width:100%;
		height:auto;
		border:solid 3px #CCC"/>

Justifying text

The template had the text automatically set to left aligned text. I thought it would look cleaner to make the text justified. To do so I changed the code to the following in the main.css file. 

div {
	text-align: justify;
	text-justify: inter-word;
	}

Displaying Code

I had some trouble with displaying html code. I got all the information I needed on this website . I used the following code: 

<pre>&lgt;code> ...... </code> </pre>

Adding changes to the website

Over the last couple of weeks I've been making gradual changes to the website in a local file on my computer. Instead of committing and pushing the changes to git consistantly, I decided to commit and push everything at the same time when I'd finished the projects. To upload the projects to Git I wrote the following in GitBash: 

git add .
git commit -m 'description of what changes you made'
git push
When I did that I got an error. I couldn't push the changes because I had accidentally saved a video in the local folder which was larger than the maximum allowed file size. Even though I deleted the video, the error persisted because it was still in the history of the repository. I tried googling this problem but nothing I found seemed to work. In the end I got a bit frustrated with the whole thing and found an article online talked about how to delete the changes that had been made to the repository since the plast git push. I decided to follow their instructions which led me to hard reset the whole repository, meaning all of the changes I had made got deleted from my local folder, even the pictures I had used were gone from my computer. I realized how extremely stupid it was to hard reset the whole thing but I just thought GitBash didn't have permission to make changes to files that were saved locally on your computer. When googling this problem it seemed like the work I had put into this was lost for good. Just before I started writing everything again I decided to talked to Hafliði about this issue, which I am very glad I did. After some googling he was able to recover all of the files I had lost, here you can see the steps we had to take to recover the files.

LESSON LEARNED: Don't hard reset the repository even though there are some things that you want to delete from it. I also figured out that it is a good idea to keep a copy of the files in another folder that is not directly connected to the repository just in case.
